# 升级AI评估指标体系 v1.0

> **设计原则**：升级的核心目标不是"赢轮"，而是"得分/控分"。赢轮只是手段，得分才是目的。
> 庄家目标：把闲家得分压到<80分；闲家目标：得分≥80分夺庄。

---

## 一、指标设计理念

传统评估用"保墩率""赢轮率"衡量AI出牌质量，但这些指标有根本缺陷：

1. **赢轮≠得分**：无分牌的轮赢了只获得出牌权，不直接得分
2. **赢轮有成本**：用大王赢一轮无分牌是浪费，不如留着吃5/10/K
3. **出牌权价值取决于后续**：赢轮后领打能得分才是有效出牌权
4. **手牌结构影响后续多轮**：出副对可能拆散连对，清门可能创造毙牌机会

因此评估应围绕**"每步出牌对最终分数的贡献"**展开，分四层：

```
第一层：轮次得分价值 —— 这轮有没有得分/丢分？
第二层：出牌权链式价值 —— 赢轮后下一轮能不能得分？
第三层：手牌结构价值 —— 出牌后手牌变好了还是变差了？
第四层：全局战略价值 —— 整局节奏、扣底控制等
```

---

## 二、评估指标详解

### A. 轮次得分价值（Round Score Value）

核心思想：**每一轮的价值=该轮的分数流动**。

| 编号 | 指标名 | 定义 | 庄家视角 | 闲家视角 |
|------|--------|------|---------|---------|
| A1 | **轮均得分** | 己方总得分 / 赢轮数 | 越低越好（控分） | 越高越好（跑分） |
| A2 | **有分轮赢率** | 场上有≥5分时己方赢轮率 | 越高越好（控住分） | 越高越好（抢到分） |
| A3 | **无分轮赢率** | 场上0分时己方赢轮率 | 参考（出牌权价值） | 参考 |
| A4 | **大分轮赢率** | 场上≥10分时己方赢轮率 | **最关键**指标 | **最关键**指标 |
| A5 | **分牌贴送率** | 队友大时己方出分牌的比例 | — | 体现配合意识 |
| A6 | **分牌损失率** | 己方分牌被对手赢走的比例 | 越低越好 | 越低越好 |

**计算方法**：
- 每轮4人出完→用compare_outcards判赢家→统计本轮出牌中5/10/K的总分
- 闲家赢→闲家得分+该分数；庄家赢→分数归零（闲家不得）
- "有分轮"：本轮4人出的牌中包含至少一张5/10/K
- "大分轮"：本轮包含≥10分（即至少一张10或K）

### B. 出牌权链式价值（Lead Chain Value）

核心思想：**赢轮获得出牌权，出牌权的价值在于下一轮能不能得分**。

| 编号 | 指标名 | 定义 | 说明 |
|------|--------|------|------|
| B1 | **赢轮→领打得分率** | 赢轮后作为领打者下一轮己方得分的概率 | 直接衡量出牌权价值 |
| B2 | **得分链长度** | 连续赢轮且连续得分的平均轮数 | 衡量"滚雪球"能力 |
| B3 | **出牌权成本** | 赢轮所用牌的平均价值(rank+是否主牌) | 用大牌赢小轮是浪费 |
| B4 | **出牌权收益率** | 赢轮后所得分数 / 赢轮所用牌的价值 | 核心效率指标 |
| B5 | **出牌权浪费率** | 赢轮后下一轮对手得分的概率 | 出牌权未能转化为得分 |

**计算方法**：
- 按round_num排序，找连续轮次中己方赢轮的序列
- B1: 赢轮r后，r+1轮己方得分的次数/赢轮总次数
- B2: 连续赢轮且得分的序列长度（如赢3轮得3轮=长度3）
- B3: 赢轮者的出牌rank均值（主牌rank按get_zhu_rank归一化到0-15）
- B4: (下一轮得分) / (赢轮者出牌rank均值)
- B5: 赢轮r后，r+1轮对手得分的次数/赢轮总次数

### C. 手牌结构价值（Hand Structure Value）

核心思想：**好的出牌不仅要看当下，还要改善/恶化手牌结构**。

| 编号 | 指标名 | 定义 | 说明 |
|------|--------|------|------|
| C1 | **绝门创造率** | 出牌后某花色副牌清零的轮数占比 | 绝门后可毙牌，战术价值极高 |
| C2 | **绝门→毙牌得分率** | 绝门后在跟牌时出主牌赢轮得分的比例 | 衡量绝门的实际收益 |
| C3 | **副对拆连对损失率** | 领打副对时该对子属于更大连对的比例 | 拆散连对是负价值 |
| C4 | **对手大牌消耗率** | 己方出牌后迫使对手出大牌(rank≥10)跟牌的比例 | 消耗对手资源 |
| C5 | **清门效率** | 每清掉一个花色副牌，后续净得分的期望 | 清门的投入产出比 |

**计算方法**：
- C1-C5需要手牌信息，现有数据不包含
- 短期用代理指标：绝门≈跟牌时出主牌（非首出主牌）
- 长期需修改数据收集脚本，记录每步出牌前的手牌快照

### D. 全局战略价值（Strategic Value）

核心思想：**不同阶段得分价值不同，关键轮次价值更高**。

| 编号 | 指标名 | 定义 | 说明 |
|------|--------|------|------|
| D1 | **前期得分占比** | 前1/3轮的得分占总分比例 | 了解节奏分布 |
| D2 | **中期得分占比** | 中1/3轮的得分占比 | — |
| D3 | **后期得分占比** | 后1/3轮的得分占比 | 后期得分更关键(扣底风险) |
| D4 | **扣底控制率** | 做庄时最后一轮不被闲家赢的比例 | 扣底可能翻倍丢分 |
| D5 | **80分临界胜率** | 闲家得分在60-100分区间时己方赢的概率 | 关键区间的把控力 |
| D6 | **节奏一致性** | 得分在时间轴上的方差（均匀vs突发） | 均匀得分更稳定 |

### E. 配合效率（Team Coordination）

核心思想：**升级是2v2游戏，队友配合效率直接影响胜负**。

| 编号 | 指标名 | 定义 | 说明 |
|------|--------|------|------|
| E1 | **贴分成功率** | 队友大时己方贴分牌且分数最终留在己方的比例 | 核心配合指标 |
| E2 | **绝门配合率** | 一人绝门+队友出该花色→绝门者毙牌得分的轮数 | 主动创造毙牌机会 |
| E3 | **信息泄露损失** | 领打对子暴露了对子信息→对手后续利用的损失 | 领打策略的信息代价 |
| E4 | **队友保护率** | 己方出分牌时队友有足够大牌保护的比例 | 反映手牌管理 |

## 三、评估方案实施

### Phase 1：现有数据可计算的指标（A/B/D类）

修改现有分析脚本，从500局数据中计算：

- **A类**：按轮次统计得分，区分有分/无分/大分轮
- **B类**：追踪轮次链，计算出牌权收益
- **D类**：按阶段统计得分分布

### Phase 2：增强数据收集

修改`run_az_vs_heuristic.py`，增加记录：
- 每步出牌前的手牌快照（hand_snapshot）
- 每步出牌前的手牌结构分析（对子/连对/绝门状态）
- 每轮结算时的场上分数（epoch_score）

### Phase 3：C/E类指标计算

基于Phase 2收集的手牌信息，计算：
- 绝门创造率、绝门利用效率
- 对子拆散损失
- 对手大牌消耗率
- 配合效率指标

### Phase 4：对比评估框架

建立标准化的AI对比评估框架：
```python
def evaluate_ai(games_data, ai_team_seats):
    """计算所有指标，返回评估报告"""
    report = {}
    report['A'] = calc_round_score_value(games_data, ai_team_seats)
    report['B'] = calc_lead_chain_value(games_data, ai_team_seats)
    report['C'] = calc_structure_value(games_data, ai_team_seats)  # 需Phase 2
    report['D'] = calc_strategic_value(games_data, ai_team_seats)
    report['E'] = calc_coordination_value(games_data, ai_team_seats)  # 需Phase 2
    return report
```

## 四、指标权重与综合评分

### 庄家综合评分
```
庄家分 = A2×0.25 + A4×0.30 + B1×0.15 + B4×0.10 + D4×0.15 + E1×0.05
```
（侧重控分和扣底控制）

### 闲家综合评分
```
闲家分 = A2×0.20 + A4×0.25 + B1×0.20 + B2×0.15 + C1×0.10 + E1×0.10
```
（侧重跑分和得分链）

> 权重可后续根据与胜率的相关性做回归分析校准。

## 五、与胜率的相关性验证

最终验证方式：**每个指标与实际胜率的Pearson/Spearman相关系数**。

- 跨100+局做per-game指标计算
- 每局得到一组合指标值+胜负标签
- 计算各指标与胜负的相关性
- 相关性最高的指标权重提升

这样评估指标本身也有数据驱动的校准闭环。
